【 프로젝트 구성 파일 】

생성된 파일 목록:

1. GameManager.cs
   • 게임 전체 상태 관리 (AppClick → Login → Algorithm → Reels → MiniGame → Ending)
   • 배터리 시스템 (100% → 0%, 1%에서 게임 종료)
   • 플레이어 통계 (저장한 수, 하트, 사용 시간)
   • 역할: 게임의 중심 뼈대 (가장 중요!)

2. UIManager.cs
   • 모든 UI 화면의 활성/비활성화 관리
   • GameManager의 상태 변화에 따라 화면 전환
   • 역할: UI 화면 전환의 중간 다리 역할

3. AppClickScreen.cs
   • 📱 앱 클릭 화면 (게임 시작 화면)
   • 앱 아이콘 클릭 → 로그인 화면으로
   • 역할: 게임 진입점

4. LoginScreen.cs
   • 🔐 로그인 화면 (현재까지의 통계 표시)
   • 저장한 수, 하트 수, 사용 시간 표시
   • 로그인 버튼 클릭 → 테마 선택 화면으로
   • 역할: 플레이어의 행동 누적 표시

5. AlgorithmScreen.cs
   • 🎨 테마 선택 화면
   • 4개의 테마 (일상, 게임, 엔터테인먼트, 교육)
   • 테마 선택 → 릴스 화면으로, 배터리 100% 초기화
   • 역할: 게임 본편 시작 전 선택 화면

6. ReelsScreen.cs
   • 📺 릴스 화면 (SNS 피드, 게임의 핵심 플레이 공간)
   • 게시물 6~8개 동적 생성 (30% 확률로 미니게임 포함)
   • 배터리 실시간 표시 (색상 변화)
   • 새로고침 기능
   • 역할: 실제 플레이가 일어나는 주무대

7. MiniGameScreen.cs
   • 🎮 미니게임 화면 (임시 화면, 나중에 확장 가능)
   • 5가지 미니게임 중 랜덤 선택 (좋아요 연타, 타이밍 맞추기, 야바위, 퀴즈, 숨은그림찾기)
   • 완료 또는 스킵 → 배터리 10% 감소, 릴스로 복귀
   • 역할: 배터리를 소모시키는 게임 기전

8. EndingScreen.cs
   • 💀 엔딩 화면 (배터리 1% 도달 시)
   • 검은 배경 + 실루엣 + 메시지
   • 최종 통계 표시
   • 재시작 버튼
   • 역할: 게임 종료 및 메시지 전달

+2개의 가이드 문서:
   • 00_프로젝트_전체_구조_가이드.txt
   • 01_Unity씬셋업_가이드.txt
   • 02_빠른참조가이드.txt

【 게임 흐름 정리 】
================================================================================

Step 1: 📱 앱 클릭
   └─ AppClickScreen 표시
   └─ 사용자가 앱 아이콘 클릭

Step 2: 🔐 로그인
   └─ LoginScreen 표시
   └─ 통계 표시: 저장한 수, 하트, 사용 시간
   └─ 사용자가 로그인 버튼 클릭

Step 3: 🎨 테마 선택
   └─ AlgorithmScreen 표시
   └─ 4가지 테마 표시
   └─ 사용자가 테마 선택

Step 4: 📺 릴스 화면 (게임 본편 시작)
   └─ ReelsScreen 표시
   └─ 배터리 100% 초기화
   └─ 게시물 6~8개 생성 (일반 + 미니게임)
   
Step 4-1: 일반 게시물 클릭
   └─ 저장한 수 +1
   └─ 다시 릴스로
   
Step 4-2: 미니게임 게시물 클릭 (30% 확률)
   └─ MiniGameScreen 표시
   └─ Step 5로

Step 5: 🎮 미니게임
   └─ 5가지 미니게임 중 1개 랜덤 선택
   └─ 완료 또는 스킵 버튼 클릭

Step 6: 배터리 감소
   └─ 배터리 -= 10%
   └─ 하트 +1
   └─ 시간 +5초
   └─ if (배터리 > 1%) → Step 4로 (릴스 복귀)
   └─ else → Step 7로 (엔딩)

Step 7: 💀 엔딩 화면
   └─ 검은 배경 + 실루엣
   └─ "배터리가 모두 소모되었습니다" 메시지
   └─ 최종 통계 표시
   └─ 재시작 버튼
   └─ 클릭 시 게임 처음부터

【 핵심 메커니즘 3가지 】
================================================================================

1️⃣ 상태 관리 (State Management)
━━━━━━━━━━━━━━━━━━
  GameManager에 게임 상태가 저장됨
  • 현재 상태: currentState
  • 상태 변경 함수: ChangeState()
  • UIManager가 상태에 따라 화면 표시

  이를 통해:
  - 한 번에 한 화면만 표시 (혼란 없음)
  - 게임 흐름이 일렬로 진행 (제어 가능)
  - 나중에 상태 추가 가능 (확장성)

2️⃣ 배터리 시스템 (Battery System)
━━━━━━━━━━━━━━━━━━
  GameManager에 배터리 수치 저장
  • 초기값: 100%
  • 감소 시점: 미니게임 완료 후
  • 감소량: -10%
  • 종료 조건: 1% 이하
  
  이를 통해:
  - 게임 진행 시간 시각화
  - 게임 종료 시점 자동 결정
  - 플레이어가 "시간 낭비" 직관적으로 인식

3️⃣ 이벤트 체이닝 (Event Chaining)
━━━━━━━━━━━━━━━━━
  각 화면에서 이벤트 발생 → GameManager 호출 → 상태 변경 → 화면 전환
  
  예시:
    AppClickScreen.OnAppButtonClicked()
      ↓
    GameManager.OnAppClicked()
      ↓
    ChangeState(GameState.Login)
      ↓
    UIManager.ShowScreen(GameState.Login)
      ↓
    LoginScreen 표시

  • 새로운 미니게임 추가 가능 (MiniGameScreen.SelectRandomMiniGame())
  • 새로운 상태 추가 가능 (enum에 추가)
  • 새로운 통계 추가 가능 (GameManager에 변수 추가)
  • 사운드 추가 가능 (각 이벤트 발생 시점에 추가)

✗ 개선 여지:
  • 현재 게시물 프리팹이 완전하지 않음 (나중에 개선)
  • 미니게임 로직이 다음 파일에 들어갈 예정
  • 사운드 시스템 미포함 (나중에 추가)

【 배터리 시스템 상세 】
================================================================================

배터리 초기화
  언제: 테마 선택 시 (AlgorithmScreen → Reels 전환)
  값: 100%
  함수: GameManager.OnThemeSelected()

배터리 감소
  언제: 미니게임 완료 시
  감소량: -10%
  함수: GameManager.OnMiniGameComplete()
  코드: battery -= 10f;

배터리 확인
  언제: 미니게임 완료 직후
  조건: if (battery <= 1f)
  결과: OnBatteryDepleted() 호출 → 엔딩

배터리 표시
  함수: ReelsScreen.UpdateBattery()
  표시형식: "🔋 X.X%"
  색상:
    • 50% 초과: 초록색 (정상)
    • 20~50%: 노란색 (경고)
    • 20% 이하: 빨간색 (위험)

배터리 접근
  GameManager.Instance.Battery 로 언제든 조회 가능

【 게임 설계 철학 】
================================================================================

이 게임의 핵심 메시지: "SNS 중독의 체험"

1. 반복성 (Repetition)
   └─ 게시물 클릭 → 미니게임 → 배터리 감소 → 다시 게시물
   └─ 끝없는 반복이 SNS 중독 표현

2. 자극 즉시성 (Immediate Gratification)
   └─ 게시물 클릭 → 즉시 반응
   └─ 미니게임 → 빠른 보상
   └─ SNS의 즉각적 피드백 재현

3. 자원 소진 (Resource Depletion)
   └─ 배터리가 계속 감소
   └─ 0이 되면 게임 종료
   └─ 시간과 에너지 낭비 시각화

4. 통계 누적 (Statistics Accumulation)
   └─ 저장한 수, 하트, 시간이 계속 증가
   └─ "얼마나 많이 했나" 인식하게 함

5. 엔딩 메시지 (Message at the End)
   └─ "당신은 얼마나 많은 시간과 집중력을 SNS에 소비했을까?"
   └─ 비판적 사고 유도
