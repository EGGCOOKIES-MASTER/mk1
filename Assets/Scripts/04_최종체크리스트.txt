【 GameManager.cs 】
┌─ 역할: 게임의 중심 뼈대
├─ 기능:
│  • 게임 상태 관리 (6가지: AppClick, Login, Algorithm, Reels, MiniGame, Ending)
│  • 배터리 시스템 (100% → 0% → 게임 종료)
│  • 플레이어 통계 (저장한 수, 하트, 사용 시간)
│  • 이벤트 제어 (OnAppClicked, OnLoginComplete, OnMiniGameComplete 등)
└─ 핵심: 모든 화면이 이 매니저를 통해 통신

【 UIManager.cs 】
┌─ 역할: UI 화면 관리자
├─ 기능:
│  • 6개의 화면 참조 보유
│  • 상태에 따라 올바른 화면 표시
│  • 기존 화면 비활성화, 새 화면 활성화
└─ 핵심: GameManager와 각 화면 사이의 중간 다리 역할

【 AppClickScreen.cs 】
┌─ 역할: 📱 게임 시작 화면
├─ UI 요소:
│  • 게임 제목: "1%\nSNS 중독 시뮬레이터"
│  • 앱 아이콘 버튼
└─ 동작: 앱 버튼 클릭 → GameManager.OnAppClicked() → 로그인 화면

【 LoginScreen.cs 】
┌─ 역할: 🔐 로그인 화면 (통계 표시)
├─ UI 요소:
│  • 📌 저장한 수: GameManager.SavedCount
│  • ❤️ 하트 수: GameManager.HeartCount
│  • ⏱️ 사용 시간: GameManager.TotalPlayTime
│  • 👤 프로필 이미지
│  • 🔓 로그인 버튼
└─ 동작: 로그인 버튼 클릭 → 테마 선택 화면

【 AlgorithmScreen.cs 】
┌─ 역할: 🎨 테마 선택 화면
├─ UI 요소:
│  • 📱 일상
│  • 🎮 게임
│  • 🎬 엔터테인먼트
│  • 📚 교육
└─ 동작: 테마 선택 → 배터리 100% 초기화 → 릴스 화면

【 ReelsScreen.cs 】 ⭐ 가장 중요한 화면
┌─ 역할: 📺 메인 플레이 화면 (SNS 피드)

│  • ScrollRect (수직 스크롤)
│  • 게시물 6~8개 (동적 생성)
│  • 🔄 새로고침 버튼

【 🔋 배터리 시스템 상세 】
================================================================================

초기화 시점: 테마 선택 (AlgorithmScreen → Reels)
초기값: 100%

감소 시점: 미니게임 완료
감소량: -10%
예시:
  1차: 100% → 90%
  2차: 90% → 80%
  3차: 80% → 70%
  ...
  10차: 10% → 0%
  11차 시도: 0% → 1% (최소값) → 엔딩!

디스플레이 (ReelsScreen):
  "🔋 X.X%" 형식
  색상:
    • 50% 초과: 🟢 초록색 (정상)
    • 20~50%: 🟡 노란색 (경고)
    • 20% 이하: 🔴 빨간색 (위험)

엔딩 조건:
  if (battery <= 1f)
  {
    GameManager.OnBatteryDepleted();
    ChangeState(GameState.Ending);
  }

【 📊 통계 시스템 】
================================================================================

저장한 수 (SavedCount)
  증가 시점: 일반 게시물 클릭
  증가량: +1
  표시: LoginScreen에서 "📌 저장한 수: X"

하트 수 (HeartCount)
  증가 시점: 미니게임 완료
  증가량: +1
  표시: LoginScreen에서 "❤️ 하트: X"

사용 시간 (TotalPlayTime)
  증가 시점: 미니게임 완료
  증가량: +5초
  표시: LoginScreen에서 "⏱️ 사용 시간: X분 Y초"

이 통계들은:
  ✓ 게임 실행 중 누적
  ✓ 로그인 화면에서 표시
  ✓ 엔딩 화면에서 최종 통계로 표시

【 🔌 코드 구조의 핵심 3가지 】
================================================================================

【 1. 상태 관리 (State Management)】
━━━━━━━━━━━━━━━━━━━━━━━━
GameManager가 현재 상태를 enum으로 관리
  enum GameState
  {
    AppClick,    // 0
    Login,       // 1
    Algorithm,   // 2
    Reels,       // 3
    MiniGame,    // 4
    Ending       // 5
  }

어느 시점에나 하나의 상태만 활성화
→ 흐름이 명확하고 제어하기 쉬움

【 2. 배터리 시스템 (Battery System)】
━━━━━━━━━━━━━━━━━━━━━━━━
플레이어의 행동을 자원으로 수치화
  • 배터리 = 제한된 자원
  • 미니게임 = 자원 소비
  • 0% = 게임 종료

→ SNS 중독의 시간/에너지 소모를 시각화

【 3. 이벤트 체이닝 (Event Chaining)】
━━━━━━━━━━━━━━━━━━━━━━
각 화면에서 발생한 이벤트 → GameManager 알림 → 상태 변경 → 화면 전환

예시:
  AppClickScreen.OnAppButtonClicked()
  → GameManager.OnAppClicked()
  → ChangeState(GameState.Login)
  → UIManager.ShowScreen(GameState.Login)
  → LoginScreen 표시

→ 모든 로직이 일관된 흐름으로 진행
